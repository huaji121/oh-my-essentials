# 🧬 Oh My Essentials — 3D Game of Life

在 Minecraft Bedrock Edition 中运行的三维元胞自动机。使用自定义方块模拟任意规则的三维生命游戏，支持持久化存储、多维度独立运行、自动步进和规则热切换。

---

## 功能

- **三维生命游戏**：基于26邻居的三维元胞自动机，支持任意存活/诞生规则
- **多维度支持**：Overworld、Nether、The End 及自定义维度各自独立运行
- **持久化**：细胞状态和规则通过动态属性跨存档保存
- **自动步进**：可指定步数和间隔 tick 自动运行
- **性能优化**：使用 `system.runJob` 分帧执行，使用坐标打包替代哈希避免对象分配，未加载区块自动跳过

---

## 安装

1. 输入yarn local-deploy
2. 在世界设置中启用该行为包
3. 开启 **Beta APIs** 实验性功能

> 需要 Minecraft Bedrock Edition 26.10 或更高版本

---

## 使用方法

### 放置和删除细胞

直接在游戏中放置或破坏 `ome:cell` 方块即可。细胞状态会自动同步并持久化。

---

### 指令列表

所有指令通过 `/scriptevent` 发送。

#### 手动步进

```
/scriptevent cell:step
```

对所有维度中有细胞的区域执行一次步进。

```
/scriptevent cell:step_dim minecraft:overworld
```

只对指定维度执行一次步进。

---

#### 自动步进

```
/scriptevent cell:run <次数> <间隔ticks>
```

示例：每 20 tick 步进一次，共跑 100 步：

```
/scriptevent cell:run 100 20
```

每步结束后会输出剩余步数，全部完成后输出"自动运行完成"。

---

#### 终止运行

发送 `/scriptevent cell:stop` 可随时终止自动运行。当前正在执行的步进会跑完后停止。

---

#### 生成随机细胞

/scriptevent cell:fill <半径> <密度> [噪声缩放] [seed]
以当前玩家位置为中心，在球形范围内使用三维柏林噪声生成随机细胞。

| 参数       | 说明                           | 默认值 |
| ---------- | ------------------------------ | ------ |
| `半径`     | 球形范围半径（格）             | 必填   |
| `密度`     | 细胞密度，0~1 之间             | 必填   |
| `噪声缩放` | 噪声频率，越小越大块           | `0.1`  |
| `seed`     | 随机种子，相同种子生成相同图案 | 随机   |

示例：
/scriptevent cell:fill 20 0.4
/scriptevent cell:fill 30 0.5 0.05
/scriptevent cell:fill 15 0.6 0.2 12345

噪声缩放参考：

- `0.05`：大块团状，结构连续平滑
- `0.1`：中等颗粒感（默认）
- `0.3`：细碎，接近纯随机

---

#### 切换规则

```
/scriptevent cell:rule <诞生条件>/<存活条件>
```

格式为逗号分隔的邻居数，诞生和存活用 `/` 分隔。规则会立即生效并持久化保存。

示例：

| 指令                             | 规则名      | 说明                      |
| -------------------------------- | ----------- | ------------------------- |
| `/scriptevent cell:rule 6/5,6,7` | **B6/S567** | 默认规则，Conway 三维类比 |
| `/scriptevent cell:rule 5/4,5`   | **B5/S45**  | Bays 1987年经典规则       |
| `/scriptevent cell:rule 3/4,5`   | **B3/S45**  | 会生长出珊瑚状枝丫结构    |
| `/scriptevent cell:rule 5/4`     | **B5/S4**   | 极稀疏，生长出针状结构    |
| `/scriptevent cell:rule 5/8`     | **B5/S8**   | 极对称，像晶体一样生长    |
| `/scriptevent cell:rule 3/2,3`   | **B3/S23**  | Conway 原版规则移植到三维 |

---

## 规则说明

规则格式为 `B<诞生>/S<存活>`，其中：

- **诞生（B）**：当前死细胞在邻居数满足条件时变为活细胞
- **存活（S）**：当前活细胞在邻居数满足条件时存活，否则死亡
- 邻居数范围为 0–26（三维26邻居）

例如 `B6/S5,6,7` 表示：死细胞恰好有 6 个活邻居时诞生；活细胞有 5、6 或 7 个活邻居时存活。

---

### 步进流程

每次步进分为4个阶段，通过 `system.runJob` 分散到多个 tick：

1. 遍历活细胞，累加26个方向的邻居计数
2. 根据规则计算下一代
3. 清除死亡方块
4. 放置新生方块

### 持久化

`JsonStore<T>` 将任意 JSON 数据分片存入动态属性（每片 ≤30,000 字节），自动处理分片数量变化。细胞坐标以 `[x, y, z][]` 格式压缩存储。
